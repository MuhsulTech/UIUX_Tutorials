Foundations of UI/UX for the web

The goal of this module is to make you think like a UX designer, not just a coder. By the end, you’ll understand what UX and UI actually are, how they connect, and how your frontend skills translate design decisions into real user experiences.

Conceptual thinking User goals Business goals Flows & structure
UX: The invisible experience

What UX designers really care about

UX is everything a user feels before, during, and after using your website. It’s not colors and fonts. It’s clarity, flow, and how easy it is to reach a goal.

  • ①
    User research: understanding who your users are, what they need, and what frustrates them.
  • ②
    User goals vs business goals: helping users succeed while still serving the product or company.
  • ③
    User flows: mapping the steps users take to complete a task (e.g., “visit → view project → contact you”).
  • ④
    Information architecture: how pages and content are organized so users never feel lost.
  • ⑤
    Wireframes & prototypes: planning layouts and flows before going into visual details.
UX is about decisions UX starts before the UI
UI · UX · Frontend

How everything connects for you

You can think of UX, UI, and frontend development as layers of the same product. Each layer answers a different question.

UX “What should exist?” Flows, structure, tasks, user journeys.
UI “How should it look?” Layout, color, typography, visual hierarchy.
Frontend “How does it behave?” Code, interactions, performance, responsiveness.
💡
Think like this as a web developer

Instead of “What component should I code?”, ask: “What problem is this screen solving for the user?” Code becomes the implementation of UX decisions, not just visuals.

1. What is UX (User Experience)?

UX is the total experience someone has while interacting with your product. It begins before they load your page (expectations) and continues after they close it (satisfaction, frustration, or delight).

Key elements of UX for web
You’ll apply these in almost every project.
  • ◎
    Clarity: is it instantly obvious what this page is about and what to do next?
  • ◎
    Efficiency: how many steps does it take to complete a task? Can we reduce them?
  • ◎
    Feedback: does the interface respond clearly when users click, submit, or navigate?
  • ◎
    Error handling: what happens if something goes wrong? Do we guide the user or blame them?
UX deliverables
What UX designers actually produce.
  • ✎
    Personas: fictional but realistic descriptions of your target users.
  • ✎
    User flows: diagrams showing paths users take to achieve goals.
  • ✎
    Wireframes: low‑fidelity screens focusing on layout and flow.
  • ✎
    Prototypes: clickable experiences to test ideas before building them.
👀
Mental model for UX

Think of UX as removing friction. Every time a user hesitates, is confused, or doesn’t know what to click — there’s a UX problem waiting for you to solve it.

2. What is UI (User Interface)?

If UX is the invisible experience, UI is the visible layer that users actually see and touch: layout, colors, typography, contrast, spacing, and all visual elements.

UI focuses on visuals & hierarchy
  • ◆
    Layout: how content is arranged on the page (grids, sections, columns).
  • ◆
    Color: choosing a palette that supports brand and usability.
  • ◆
    Typography: selecting fonts, sizes, and weights for readability and emphasis.
  • ◆
    Spacing: using white space so the interface feels calm, not crowded.
Common UI components on the web
  • ◼
    Buttons: primary, secondary, ghost, icon buttons.
  • ◼
    Cards: for projects, articles, or products.
  • ◼
    Forms: inputs, labels, error states, success states.
  • ◼
    Navigation: headers, sidebars, footers, mobile menus.
🎨
UI is not just “making it pretty”

Strong UI supports UX. Good visual hierarchy guides the eye. A clear primary button reduces confusion. Clean spacing makes it easier to scan.

3. UX vs UI vs Frontend Development

As a frontend developer, you sit at the bridge between design and reality. UX and UI define the “plan”; your code makes it exist and feel responsive and alive.

Same problem, three perspectives
  • 1
    UX asks: “What is the user trying to achieve on this screen?”
  • 2
    UI asks: “How can we show this in a clear, attractive way?”
  • 3
    Frontend asks: “How can I build this so it’s fast, accessible, and works on all devices?”
Your advantage as a hybrid

If you understand UX and UI, you don’t just “code what you’re given”. You can improve flows, suggest better layouts, and make small UX improvements directly in your implementation.

For example, you might add smart validation messages, focus states, keyboard-friendly navigation, or subtle animations that clarify what’s happening.

🧠 Code that respects UX decisions feels 10× more professional.

4. The UX mindset for web developers

Before we touch tools like Figma, the most important upgrade is your mindset. UI/UX is not a collection of tricks — it’s a way of thinking about users and problems.

From “how do I build this?” to “why does this exist?”
  • ?
    Ask “why” first: why is this page here? What user problem does it solve?
  • ?
    Define success: how will we know this page works? (e.g., contact form submissions, time on page).
  • ?
    Remove friction: where might users get stuck, confused, or bored?
You are not the user

You know your website too well. You know every button, every shortcut. Real users don’t. A UX mindset means assuming that:

  • ★
    People scan, they don’t read every word.
  • ★
    People are impatient; if they get stuck, they leave.
  • ★
    People don’t care about features; they care about outcomes.

5. Practical exercise: Login → Dashboard flow

To make this concrete, you’ll design a simple user flow for a “Login → Dashboard” experience and think about UX before UI.

✅
Exercise · 20–30 minutes
  1. Write down the user goal in one sentence, e.g.: “A returning user wants to log in quickly and see their dashboard.”
  2. On paper or any drawing tool, sketch the user flow steps:
    – Open website →
    – Click “Login” →
    – See login form →
    – Enter email & password →
    – Submit →
    – See success or error →
    – Reach dashboard.
  3. Under each step, write one sentence about what could make it confusing or frustrating for the user.
  4. Next to that, write one UX improvement idea for each step (e.g., “show password strength”, “clear error messages”, “remember email”, “loading indicator”).
🧩
Optional: connect this to your portfolio

Repeat the same process for your own portfolio: “Home → View Project → Contact”. You’ll reuse this in later modules when we design your actual UI and code it.
